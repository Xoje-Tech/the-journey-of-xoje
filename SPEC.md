# SPEC — Slice 1 · The Journey of Xoje

> Status: **proposed** · awaiting user OK before implementation.
> Last updated: 2026-07-14.

## Goal (one sentence)

A static, KISS portfolio website whose print output **is** the Harvard-style CV that `@personal-brand/cv` already produces, in two locales (es + en), with zero backend and zero runtime JS.

## Non-goals (explicit)

- No animations, no transitions, no dark mode toggle UI
- No WebMCP / browser native tool calling in this slice
- No SEO advanced, no sitemap, no Open Graph dynamic
- No deploy / domain / CI in this slice — the deliverable is `pnpm dev` working locally and `pnpm build` producing `dist/`
- No "videojuego portfolio", no AI-rendered UI — those belong to slice 2+

## Source of truth for content

- `~/projects/personal-brand` is the canonical source for the CV data
- `~/.personal-brand/DATA/*.json` holds the live data the MCP server reads
- The portfolio's build script **invokes `@personal-brand/cv` directly** as a workspace dependency to materialize `cv.es.md` and `cv.en.md` at build time

## The print contract (this slice's hard guarantee)

`src/styles/print.css` MUST contain, at minimum:

```css
@page {
  size: A4;
  margin: 1.6cm 2cm; /* top/bottom 1.6cm, left/right 2cm — Harvard sheet feel */
}

@media print {
  body {
    font-family: 'Charter', 'Source Serif Pro', 'Liberation Serif', 'Times New Roman', serif;
    font-size: 10.5pt;
    line-height: 1.35;
    color: #000;
    background: #fff;
  }
  h1 {
    font-size: 22pt;
    margin: 0 0 0.3cm 0;
  }
  h2 {
    font-size: 13pt;
    margin: 0.6cm 0 0.2cm 0;
    border-bottom: 0.5pt solid #444;
    padding-bottom: 0.1cm;
  }
  h3 {
    font-size: 11.5pt;
    margin: 0.35cm 0 0.1cm 0;
  }
  section[data-cv-section] {
    page-break-inside: avoid;
  }
  .no-print {
    display: none !important;
  }
  a[href]::after {
    content: '';
  } /* URLs OFF in print */
}
```

`tests/print-contract.test.ts` enforces these rules exist as text in the emitted CSS (the only deterministic check without a headless browser).

## Build flow

1. `pnpm install` — installs Astro + peer deps, links `../personal-brand` via workspace
2. `node scripts/build-cv.mjs` — reads `~/.personal-brand/DATA/*`, calls `@personal-brand/cv` `generateCv({ variant: 'cv-product-eng' })` per locale, writes `src/content/cv.{es,en}.md`
   - For EN: the script post-processes the markdown to swap the locale-dependent UI strings via the `ui.{locale}.json` dictionary (G01/G04/G05 still run over the data; no guardrail is bypassed)
3. `pnpm build` — Astro reads the markdown content collection, hydrates `[locale]/index.astro` with `CvDocument`, emits static `dist/es/index.html` + `dist/en/index.html`
4. Locale resolution: `[locale]/index.astro` checks `Astro.url.pathname`; the bare `/` route in `pages/index.astro` redirects to `defaultLocale` (`es`).

## I18n constraints

- `astro.config.mjs` declares `defaultLocale: 'es'`, `locales: ['es', 'en']`, `routing: { prefixDefaultLocale: true }` → URL shape is `/es/`, `/en/` (no bare `/cv`)
- `LocaleSwitch.astro` reads the **current** locale from `Astro.currentLocale`, builds the link to the **other** locale via `getRelativeLocaleUrl`
- UI strings live in `src/i18n/ui.{locale}.json`. The 4 keys for this slice: `site.title`, `site.tagline`, `actions.print`, `actions.switchLocale`

## Testing strategy

- **Unit test**: `tests/print-contract.test.ts` — read the emitted `dist/_astro/*.css`, assert presence of `@page`, `size: A4`, `1.6cm`, `.no-print`, `page-break-inside`.
- **Manual acceptance**: open `dist/es/index.html` in Chromium → Print Preview → confirm:
  - 1 page (when CV is short) or clean multi-page breaks on section
  - Serif font, margins 2cm L/R 1.6cm T/B
  - Header/switch nav hidden
  - URLs after links stripped
- No agentic E2E in this slice (no Playwright — that belongs to slice 2 when there's real interactivity)
